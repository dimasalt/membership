How to test your JVZoo integration with JVZIPN test data and Postman

4. March 2016 by Richard Brammer | 0 comments

To create a JVZIPN integration, you might want to test, if your service processes the data sent by JVZoo correctly.
Unfortunately JVZoo does not provide a sandbox environment to make test purchases and test your service.

Of course you may create test purchases of your product and see, if your service operates correctly.
But this process has some disadvantages:

  • it takes time to create a test purchase
  • you need a seconds Paypal account
  • each purchase cost you $0.01 (and refunding this does cost you time again)
  • if your product is already being sold, the test purchases affect your publicly visible statistics

Prerequisites

A better idea is to grab the data JVZoo sends via its JVZIPN interface once, and reuse it for testing purchases on your local or testing environment.
To follow this process, there are several prerequisites:

  • you need a host with PHP and HTTPS enabled to grab the data sent by JVZIPN once
  • you need a tool like Postman (or curl) to send POST request to your development process, using the data generated in 1

How to

The process works as following:

  1. install a PHP script on your host – it must be accessible via HTTPS
  2. Setup your product on JVZ to send the JVZIPN integration to the script’s URL
  3. Make a test purchase, the script will collect the POST data sent by JVZIPN
  4. Download the log files, that keeps the POST data
  5. Setup Postman to use the exact same data, that JVZIPN sent, to send it to your development environment
  6. After finishing your development, setup JVZIPN to post to the URL your actual integration service is hosted

Let us go through the steps one by one:

1. Install PHP script on HTTPS

Create an index.php with the content of this script and upload it to a directory of your webhost.


<?php
$date = new DateTime();

$req_dump = '=== ' . $date->format('Y-m-d H:i:s') . " ===\r\n";
$req_dump .= print_r($_POST, TRUE);
$req_dump .= print_r($_SERVER, TRUE);
$req_dump .= '=======================================================';
$fp = fopen('request.log', 'a');
fwrite($fp, $req_dump);
fclose($fp);

If you open the URL in your browser, you shouldn’t see anything, because for GET requests, the script doesn’t send anything.

2. Setup product on JVZ

Open the product via “edit”. Within the External Program Integration Tab, open use method #1 and point to the HTTPS-URL the script is hosted under.
Choose “No for output as Key Generation” (actual it doesn’t matter, if it’s yes or no). And for JVZIPN Special Integration choose “General integration”.
Save the product.

3. Test purchase to collect data

Go to the seller’s dashboard. In the bottom left beneath your product list choose “Test purchases”.
Go through the process of creating a test purchase for your product. You will need a Paypal account, that isn’t connected to your JVZoo profile – e.g. a personal account or an account of a friend.

4. Download the log file

Access your host account, where the script is hosted. a request.log file should have been created next to the index.php.
You may also access it via the URL the script is hosted, adding request.log (or replacing index.php by request.log)
Open the request.log and you will see the headers and POST data of the request, that was made by JVZIPN during your test purchase.

5. Setup Postman

Download, install and open Postman if you haven’t already. Create a new POST request pointing to the development server of the service you want to integrate with JVZoo. Now add the data fields from request.log as body fields for the POST request in Postman. the format is x-www-form-urlencoded (as you an see in variable dump of $SERVER).

POST values sent by JVZIPN

Now you may use Postman to emulate a test purchase, whenever you want.

6. Setup product at JVZoo

As soon as you are happy with your integration service, deploy it and do not forget to setup your product to point to the new integration service.
